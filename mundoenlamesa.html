Plataforma web de gestión para restauración

VER INFORME TFG MI PORTFOLIO

INTRODUCCIÓN

Este proyecto es una aplicación web dinámica desarrollada como Proyecto Final de Grado en Desarrollo de Aplicaciones Web. La plataforma está diseñada para gestionar reservas, usuarios y contenidos en negocios de restauración, ofreciendo diferentes áreas privadas según el rol de cada usuario.

EL RETO

El principal objetivo del proyecto fue desarrollar una solución web completa que centralizara la gestión de reservas y usuarios, permitiendo un control eficiente desde un único sistema y mejorando la experiencia tanto de clientes como de empleados.

FUNCIONALIDADES PRINCIPALES

Roles

Sistema de acceso con roles diferenciados: cliente, empleado y administrador

Reservas

Gestión completa de reservas (creadas, confirmadas, pasadas y canceladas)

Usuario

Área de usuario con edición de datos personales y visualización de recompensas

Fidelización

Programa de fidelización basado en reservas completadas

Empleado

Área de empleado para gestión de reservas y mensajes de contacto

Administración

Panel de administración para gestión de empleados y contenidos

Carta

Edición dinámica de la carta del restaurante desde el panel administrativo

ARQUITECTURA Y FUNCIONAMIENTO

La aplicación está estructurada en vistas públicas y áreas privadas, con control de acceso según el rol del usuario. Toda la lógica del sistema se apoya en una base de datos relacional que gestiona usuarios, reservas, empleados y contenidos, garantizando la coherencia y persistencia de la información.

BASES DE DATOS

El sistema utiliza múltiples tablas relacionadas entre sí para gestionar usuarios, reservas, empleados, mensajes y contenidos. Esta estructura permite un manejo eficiente de la información y facilita la escalabilidad del proyecto a futuro.
